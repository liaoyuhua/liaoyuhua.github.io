通过Pycharm和Github进行项目版本管理

  • 准备工作
  • 在Pycharm中配置Github和Git
  • 新建本地项目
  • 在项目地址生成Git仓库
  • 将本地项目分享到Github
  • 将后续修改提交到Github仓库中

准备工作

Github账号、Pycharm、Git

在Pycharm中配置Github和Git

在导航栏中，点击“File”，点击“Setting”：

点击左侧选项栏“Version Control”，点击子选项“Git”：

点击图示文件夹图标，找到自己安装Git的路径，选中git.exe：

点击OK，就配置好了Git。

另外找到临近“Git”选项的“Github”选项，点击进入：

按照提示添加自己的Github账号，应该会弹出到浏览器进行身份验证。

新建本地项目

在Pycharm新建了个‘Loan-Predicting’的项目，里面有一个‘compete-code.py’的python文件。注意：项目里面如果没有进行版本控制（Version Control）的话，文件是橙色的；如果是新建的进行版本控制的文件，就是绿色的；如果是已有的进行版本控制的文件，并且被修改了，未commit之前是蓝色的；如果是已有的进行版本控制的文件，没有被修改，是白色的。如下图所示：

在项目地址生成Git仓库

点击pycharm导航的“VCS”：

点击“Create Git Repository”，找到你的项目地址：

点击ok。

将本地项目分享到Github

点击导航栏的“Git”，点击子选项“Github”，点击“Share Project on Github”：

修改仓库名和选择是否建为私密仓库：

然后会弹出下面这个窗口，根据需要将自己的项目文件上传到Github中，我这里因为只有一个文件，所以只勾选一个：

右下角出现这样的信息，就说明成功了：

将后续修改提交到Github仓库中

点击导航栏的“Git”，点击“Commit”，填写相关内容，然后点击“Push”即可。和正常的Git操作逻辑一致：
